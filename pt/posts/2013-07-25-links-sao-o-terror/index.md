A World Wide Web é enorme. Repleta de links. Clica aqui. Ops, mais links. Mais
links ainda. Cliques. AaaaaAAAAaaAA.....

E se você quiser **gerenciar** e guardar alguns desses links? Quais são suas
opções? E se alguns links são arquivo morto, outros são lançadores, páginas que
você acessa com certa frequência, e se outros são parte de um trabalho, e assim
por diante? O que pode ser feito para gerenciar isso tudo? Esse post é uma
coleção de ideias que podem ser úteis para realizar um gerenciamento eficiente
(porém simples!!) de links e de referências na Web, as quais coletei e
aperfeiçoei pelo menos durante 3 anos.

Todos os itens a seguir foram usados por mim por um bom tempo; alguns eu
abandonei, outros eu utilizo até hoje, uns mais, outros menos. Essas ideias
estão acompanhadas de uma pequena descrição e mais a minha opinião / experiência
de uso.

Primeiramente: o objetivo é ser **simples**. Existem diversas soluções por aí
que são um tanto complexas. Isso só deixa a pessoa que organiza tudo maluca.
Complexidade só atrapalha. Então vamos lá:

**1 - Arquivo Morto: Delicious**

[Delicious.com](http://www.delicious.com): Um ótimo site+ferramenta para guardar
links. Criado especialmente para esse fim. Ele também tem um quê de social,
permitindo que links sejam compartilhados num esquema parecido com o do Twitter,
de seguidores e pessoas que você segue. Você pode usá-lo para criar coleções de
links, através de tags. Também é possível selecionar individualmente se os links
guardados lá serão privados ou públicos.

Opinião: é a melhor ferramenta (gratuita!) para guardar links on-line que já
usei. Existe um *bookmarklet* que facilita **muito** adicionar qualquer link
direto da barra de favoritos do seu navegador. Ponto fraco: péssimo *mobile*. Só
é bom no *desktop*.

**2 - Barra de favoritos no seu navegador + pastas + hierarquia**: é uma ótima
forma mais rápida para guardar links. Uma solução bastante eficiente, porém pode
ficar rapidamente desorganizada. A desvantagem dessa forma em relação à anterior
é que os links ficam visíveis o tempo todo, não ficam escondidos. Mesmo que vocẽ
utilize pastas. Daí, pode dar um certo *overhead*. É mais recomendado para
arquivar links que o leitor já conhece (e não links de propósito geral, por
exemplo, de uma página interessante que o leitor encontrou na web).

Se o seu navegador suporta sincronização (todos os modernos suportam
isso...opera link, firefox sync, chromium / google chrome), então você pode ter
acesso rápido inclusive *mobile*.

Para o Firefox, veja essa página: [http://support.mozilla.org/en-US/kb/use-bookmarks-to-save-and-organize-websites](http://support.mozilla.org/en-US/kb/use-bookmarks-to-save-and-organize-websites) (em inglês).

Para organização em massa, além da barra de favoritos, você pode utilizar também
o menu de favoritos do seu navegador.

**3 - Evernote**

Veja [https://www.evernote.com/](https://www.evernote.com/) . O Evernote é um
"balde" para guardar tudo. Tire suas próprias conclusões entrando no site.

Eu particularmente já tentei usá-lo várias vezes mas acho que ele tem funções
demais. Isso acaba mais atrapalhando do que ajudando. No entanto, ainda assim, é
uma ótima ferramenta. Tem suporte para mobile e para múltiplos navegadores. Tem
até clientes para desktop. E muitos sites se integram com ele.

**4 - Wiki Pessoal**

Essa solução é um pouco trabalhosa mas pode ser gratificante no final. Eu
utilizo ela para guardar parte dos meus links. A melhor forma de ver como isso
funciona é na prática. Alguns exemplos:

- http://www.florian-diesch.de/

- http://alexschroeder.ch/wiki

- http://www.stallman.org/

Uma wiki não guarda apenas links, mas ela também possui uma estrutura de auto
referência, na qual páginas dentro dela apontam para outras páginas dentro dela
(ou mesmo para fora dela). Ela é mais útil se você agrupar determinado conjunto
de informações / conhecimento de maneira organizada e com comentários / guias /
referências. Resumindo: só utilize essa opção se você quiser realmente organizar
determinado conjunto de informações. Para guardar apenas links, isso NÃO é
recomendado.

Note que nem toda wiki precisa ser on-line. Existem diversas soluções para Wikis
Off-line. Por exemplo, https://en.wikipedia.org/wiki/Wiki_software

Wikis podem ser caóticas por natureza, ou não. Isso depende de você.

**5 - Off-line Software**

Alguns softwares que funcionam off-line podem ser úteis para tomar notas,
inclusive de links. Um exemplo clássico: TomBoy
[[https://projects.gnome.org/tomboy/](https://projects.gnome.org/tomboy/)]

Ele é um software livre que permite criar várias notas e organizá-las em
categorias. Poderia ser comparado a uma miniwiki. Note que essa solução é a mais
rápida que tem, no entanto ela é bem limitada. É possível sincronizar suas notas
através de programas como o dropbox, para minimizar essa limitação.

Atualmente uso ele para guardar notas relacionadas ao desktop. De modo que notas
que eu possa precisar acessar mobile ou remotamente eu utilizo outra solução.

**6 - The Old Way: .txt files**

Na hora da preguiça, nada melhor do que criar um arquivo de texto simples e
colocar um link por linha. A desvantagem óbvia disso é que para recuperar
(procurar e encontrar) um link depois a dificuldade pode ser enorme.

Eu particularmente usava muito isso no passado, por falta de conhecimento de
outros softwares. Não recomendo mais utilizar isso como uma solução a longo
prazo, no entanto, essa é uma ótima solução temporária ou num prazo muito curto
e relativamente mais corriqueiro.

**7 - Speed Dial**

Se você usa firefox, pode checar esse complemento:
https://addons.mozilla.org/pt-br/firefox/addon/speed-dial/

O Opera suporta isso nativamente desde que eu o conheço.

Se você usa o Chrome, cheque esse aqui:
https://chrome.google.com/webstore/detail/speed-dial/dgpdioedihjhncjafcpgbbjdpbbkikmi

Speed Dial é um ótimo método para armazenar lançadores. Recomendo utilizá-lo
para os sites que você acessa diariamente ou semanalmente.

**8 - Pocket**

Existem diversos serviços para o leitor que gosta de procrastinar, gentilmente
apelidados de "read it later". O que eu mais gosto é o pocket
[[http://getpocket.com/](http://getpocket.com/)].

Funcionam assim: achou um link legal que tem bastante coisa para ler? Não quer
lê-lo agora? Guarde-o no pocket e leia depois. Esse tipo de serviço não é útil
para guardar links que não tenham conteúdos de leitura. Ele é otimizado
justamente para fins de leitura. Vale a pena ter um desses. Só não vale a pena
acumular coisa demais para ler aí...

**Conclusão**

Combinando alguns desses serviços de modo prático e eficiente pode ser uma boa
forma de se organizar.
