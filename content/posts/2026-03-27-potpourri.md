For Sprint planning I like to name my catch-all / umbrella Epics as "potpourri".

Why? [Just because](https://missing.csail.mit.edu/2020/potpourri/).

Example:

* GitOps pipeline for Terraform – potpourri
* GitOps phase potpourri – ArgoCD production wrap-up

**One advantage**: because no one else uses this nomenclature, it's very easy to
find my own tickets in JIRA this way – very important, as JIRA search [kinda
sucks]({{< ref "2025-06-10-jira-service-management-search-is-terrible" >}}).
